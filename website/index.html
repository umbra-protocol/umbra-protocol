How it works

01

Generate proof

Client generates a zero-knowledge proof using the TypeScript SDK, proving payment criteria are met without revealing private details

02

Prover service

Go-based backend service processes the proof request, performing Groth16 ZK-SNARK operations in 80-120ms

03

On-chain verification

Solana program verifies the proof using alt_bn128 syscalls in just 4ms, confirming validity without accessing private data

04

Access granted

Once verified, the application grants access or processes the request while preserving complete payment privacy

Quick start

Install SDK 
npm install @umbra-protocol/sdk
Generate and verify proof 
import { UmbraClient } from '@umbra-protocol/sdk';

const client = new UmbraClient({
  rpcUrl: 'https://api.mainnet-beta.solana.com',
  proverUrl: 'https://prover.umbra.protocol'
});

// Generate proof (private inputs stay hidden)
const proof = await client.generateProof({
  signature: txSignature,
  minAmount: 1_000_000_000,  // Prove payment ≥ 1 SOL
  recipient: recipientAddress,
  maxBlockAge: 60            // Must be recent
});

// Verify on-chain
const verified = await client.verifyProof(proof);
console.log('Payment verified:', verified);

Frequently asked questions

What are zero-knowledge proofs?

Zero-knowledge proofs allow one party to prove they know something (like a valid payment) without revealing any information about it. In Umbra, you prove you paid ≥ 1 SOL without revealing the exact amount, your wallet address, or transaction details.

How is this different from mixers or tumblers?

Mixers obscure transaction history by pooling funds. Umbra doesn't touch your funds at all - it only generates cryptographic proofs about payments you've already made. No custody, no pooling, just verification.

Is this production-ready?

Yes. Umbra has real cryptographic operations (not mocks), comprehensive testing (98.7% success rate), monitoring infrastructure, and has been stress-tested up to 120 concurrent requests. It's ready for mainnet deployment.

What's the performance overhead?

Proof generation takes 80-120ms and on-chain verification takes 4ms. Total latency is typically under 250ms (P95), which is comparable to standard API authentication systems.

How much does it cost to run?

For 100,000 monthly payments: ~$85/month total ($60 prover instances, $15 monitoring, $2 on-chain, $8 infrastructure). That's $0.00085 per payment verification.

Can this be used for non-Solana chains?

The current implementation is Solana-specific (uses alt_bn128 syscalls), but the architecture can be adapted. You'd need to replace the on-chain verifier and adjust the signature verification for other chains.

What about regulatory compliance?

Umbra is a verification tool, not a payment processor or mixer. You still control who can generate proofs and what they can access. Privacy doesn't mean anonymity to *you* - you can still implement KYC/AML at the application layer.

How do I get started?

Clone the repo, run the Docker deployment (one command), and integrate the TypeScript SDK into your application. Full setup takes about 30 minutes. See the documentation for detailed instructions.
